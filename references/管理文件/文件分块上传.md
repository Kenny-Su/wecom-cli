# 文件分块上传

**最后更新：2023/03/09**

为了支持大文件上传，可将文件进行分块后分别上传。

## 上传步骤

1. **上传初始化**
   计算文件的分块累积 `sha` 值，并提交到企业微信开放平台。
   分块大小固定为 `2M`，即 `2097152` 字节，使用 `sha1` 算法。
   如果命中秒传，后续步骤无需继续执行。

2. **分块上传文件**
   分别上传分块文件。
   可并发上传，建议并发数不超过 `10`。

3. **标记上传完成**
   如果文件上传完整，返回上传成功的 `fileid`。

---

# 权限说明

* 企业需要使用“微盘”`secret` 所获取的 `access_token` 来调用
* 自建应用需配置到“可调用应用”列表中的应用 `secret` 所获取的 `access_token` 来调用
* 第三方应用需具有“微盘”权限
* 代开发自建应用需具有“微盘”权限

# 分块上传初始化

请求分块上传初始化接口。
如果命中秒传，则流程结束，完成上传。

## 请求信息

* **请求方式**：`POST`（HTTPS）
* **请求地址**：

```text
https://qyapi.weixin.qq.com/cgi-bin/wedrive/file_upload_init?access_token=ACCESS_TOKEN
```

## 请求包体

```json
{
  "spaceid": "SPACEID",
  "fatherid": "FATHERID",
  "selected_ticket": "SELECTED_TICKET",
  "file_name": "FILE_NAME",
  "size": 123,
  "block_sha": [
    "STATE1",
    "STATE2"
  ],
  "skip_push_card": false
}
```

## 参数说明

| 参数                | 类型       | 是否必须 | 说明                                                                      |
| ----------------- | -------- | ---- | ----------------------------------------------------------------------- |
| `spaceid`         | string   | 否    | 空间 `spaceid`                                                            |
| `fatherid`        | string   | 否    | 当前目录的 `fileid`，根目录时为空间 `spaceid`                                        |
| `selected_ticket` | string   | 否    | 微盘和文件选择器 JSAPI 返回的 `selectedTicket`。若填写该参数，则无需填写 `spaceid` / `fatherid` |
| `file_name`       | string   | 是    | 文件名字                                                                    |
| `size`            | uint64   | 是    | 文件大小，最大支持 `20G`                                                         |
| `block_sha`       | string[] | 是    | 文件分块累积 `sha` 值，按分块顺序填入数组。参考“附录：分块累积 sha 说明”                             |
| `skip_push_card`  | bool     | 否    | 文件创建完成时是否推送企业微信卡片。默认 `false`，即默认推送卡片                                    |

## 注意事项

`spaceid` / `fatherid` 和 `selected_ticket` 必须填写且仅填写其中一组参数。

## 返回示例

```json
{
  "errcode": 0,
  "errmsg": "ok",
  "hit_exist": false,
  "upload_key": "UPLOAD_KEY",
  "fileid": "FILEID"
}
```

## 返回参数说明

| 参数           | 类型     | 说明                                       |
| ------------ | ------ | ---------------------------------------- |
| `errcode`    | int32  | 错误码                                      |
| `errmsg`     | string | 错误码说明                                    |
| `hit_exist`  | bool   | 是否命中秒传                                   |
| `upload_key` | string | 文件上传凭证。不命中秒传时返回，作为 `file_upload_part` 参数 |
| `fileid`     | string | 文件 `fileid`。命中秒传时返回，此时上传流程完成             |

# 分块上传文件

将文件内容按 `2M` 分块，依次请求分块上传文件接口。

## 请求信息

* **请求方式**：`POST`（HTTPS）
* **请求地址**：

```text
https://qyapi.weixin.qq.com/cgi-bin/wedrive/file_upload_part?access_token=ACCESS_TOKEN
```

## 请求包体

```json
{
  "upload_key": "UPLOAD_KEY",
  "index": 1,
  "file_base64_content": "FILE_BASE64_CONTENT"
}
```

## 参数说明

| 参数                    | 类型     | 是否必须 | 说明                                                                                          |
| --------------------- | ------ | ---- | ------------------------------------------------------------------------------------------- |
| `upload_key`          | string | 是    | 文件上传凭证。`file_upload_init` 返回的 `upload_key`                                                  |
| `index`               | int32  | 是    | 文件分块号。文件内容按 `2M` 分块，从 `1` 开始                                                                |
| `file_base64_content` | string | 是    | 分块的文件内容 Base64。只需要填入文件内容的 Base64，不需要添加类似 `"data:application/x-javascript;base64"` 的数据类型描述信息 |

## 返回示例

```json
{
  "errcode": 0,
  "errmsg": "ok"
}
```

## 返回参数说明

| 参数        | 类型     | 说明    |
| --------- | ------ | ----- |
| `errcode` | int32  | 错误码   |
| `errmsg`  | string | 错误码说明 |

# 分块上传完成

请求分块上传完成接口，流程结束，完成上传。

## 请求信息

* **请求方式**：`POST`（HTTPS）
* **请求地址**：

```text
https://qyapi.weixin.qq.com/cgi-bin/wedrive/file_upload_finish?access_token=ACCESS_TOKEN
```

## 请求包体

```json
{
  "upload_key": "UPLOAD_KEY"
}
```

## 参数说明

| 参数           | 类型     | 是否必须 | 说明                                         |
| ------------ | ------ | ---- | ------------------------------------------ |
| `upload_key` | string | 是    | 文件上传凭证。`file_upload_init` 返回的 `upload_key` |

## 返回示例

```json
{
  "errcode": 0,
  "errmsg": "ok",
  "fileid": "FILEID"
}
```

## 返回参数说明

| 参数        | 类型     | 说明          |
| --------- | ------ | ----------- |
| `errcode` | int32  | 错误码         |
| `errmsg`  | string | 错误码说明       |
| `fileid`  | string | 文件 `fileid` |

# 附录：分块累积 sha 说明

分块的累积 `sha` 值计算过程如下：

1. 将要上传的文件内容按 `2M` 分块；
2. 对每一个分块依次执行 `sha1_update`；
3. 每次 `update` 后，记录当前的 `state`，并转成 `16` 进制，作为当前块的累积 `sha` 值；
4. 当处理最后一块时，该块可能小于 `2M`。
   `update` 完成后，再执行 `sha1_final` 得到的 `sha1` 值，即整个文件的 `sha1`，作为最后一块的累积 `sha` 值；
5. 将以上过程得到的 `sha` 值按顺序放入数组，作为 `file_upload_init` 接口的 `block_sha` 参数。

相关示例代码见：`C++` 版本。其他语言可根据以上流程自行实现，并与官方提供的 `C++` 版本实际测试结果对比验证。

官方还提供了 demo 文件及其对应接口调用关键参数说明文档，可用于验证开发者逻辑实现的正确性。
