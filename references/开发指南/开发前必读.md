# 开发前必读

> 最后更新：2023/10/19

## 开发文档阅读说明

服务端 API 开放了丰富的能力接口，开发者可以借助接口能力，实现企业服务及企业微信的集成。

支持的能力可以通过目录导航快速预览，目录树按功能块聚合归类，例如：

* 通讯录管理
* 消息推送
* 等

建议的阅读顺序：

1. 先阅读开发指南
2. 阅读 `access_token` 获取方式
3. 再按需查看各功能模块文档

---

## 接口规范

所有接口均需满足以下要求：

* 使用 HTTPS 协议
* 使用 JSON 数据格式
* 使用 UTF-8 编码

接口说明格式如下：

```text
请求方式：GET/POST（HTTPS）

请求地址：
https://qyapi.weixin.qq.com/cgi-bin/gettoken?corpid=ID&corpsecret=SECRET

请求包体：
...

参数说明：
...

权限说明：
...

返回结果：
...

参数说明：
...
```

---

## 接口说明详解

### 1. 请求方式

标明接口调用的 HTTP 方法：

* `HttpGet`
* `HttpPost`

所有请求均使用 HTTPS 协议。

---

### 2. 请求地址

请求地址中的大写单词表示需要替换的变量。

例如：

```text
https://qyapi.weixin.qq.com/cgi-bin/gettoken?corpid=ID&corpsecret=SECRET
```

其中：

* `ID`
* `SECRET`

均需要替换为实际值。

假设：

```text
ID=wwabcddzxdkrsdv
SECRET=vQT_03RDVA3uE6JDASDASDAiXUvccqV8mDgLdLI
```

则最终请求地址为：

```text
https://qyapi.weixin.qq.com/cgi-bin/gettoken?corpid=wwabcddzxdkrsdv&corpsecret=vQT_03RDVA3uE6JDASDASDAiXUvccqV8mDgLdLI
```

---

### 3. 请求包体 / 参数说明

用于说明：

* 请求参数示例
* 参数字段含义
* 参数取值范围

开发者在设计数据结构时，应参考字段定义范围。

---

### 4. 权限说明

用于标明接口的使用范围。

开发者需要特别注意调用场景，例如：

* 同步通讯录接口必须使用「通讯录同步助手」的 `access_token`
* 发送消息的目标范围必须在应用可见范围内

---

### 5. 返回结果 / 参数说明

用于说明：

* 返回参数示例
* 返回字段定义

特别注意：

所有接口在调用失败时，返回包中均包含：

```json
{
  "errcode": 0,
  "errmsg": "ok"
}
```

其中：

* `errcode`

  * 存在且不为 `0` 时表示失败
  * 详细定义参考「全局错误码」

* `errmsg`

  * 仅供参考
  * 后续可能发生变化
  * 不可作为成功与否的判断依据

因此：

* `errcode == 0` → 成功
* `errcode != 0` → 失败

# 接口调用流程

## 1. 获取 access_token

参考官方文档说明获取 `access_token`。

---

## 2. 缓存与刷新 access_token

开发者需要缓存 `access_token`，用于后续接口调用。

注意事项：

* 不要频繁调用 `gettoken` 接口
* 否则会触发频率限制

当 `access_token`：

* 失效
* 过期

时，需要重新获取。

---

## 3. 调用业务接口

使用有效的 `access_token` 调用具体业务接口。

# 可信 IP

为了保障企业数据安全：

自 **2022 年 6 月 20 日 20:00** 起：

* 新开启的通讯录同步助手
* 新创建的自建应用

必须在管理端配置「可信 IP」。

只有已配置的可信 IP 才能调用接口。
