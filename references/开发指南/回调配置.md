# 回调配置

> 最后更新：2024/12/13

# 1. 什么时候需要回调服务

在集成企业微信与内部系统时，通常需要搭建一个回调服务。

回调服务可以实现以下能力：

## 1.1 自定义服务行为

例如：

* 用户向应用发送消息时，识别消息关键词，并回复不同内容
* 用户点击应用菜单时，将点击行为转化为指令，执行自动化任务

## 1.2 实时获取状态变化

例如：

* 通讯录发生变化时，无需定时拉取通讯录进行对比
* 可以实时获取变化的通讯录节点，并进行同步

---

# 2. 回调服务需要哪些配置

配置回调服务需要 3 个配置项：

* `URL`
* `Token`
* `EncodingAESKey`

---

## 2.1 URL

`URL` 是回调服务地址，由开发者自行搭建。

用途：

* 接收通知消息
* 接收事件回调

该地址需要满足接口要求，具体参考第 3 部分说明。

---

## 2.2 Token

`Token` 用于计算签名。

要求：

* 由英文或数字组成
* 长度不超过 32 位
* 由开发者自定义

由于开发者提供的 `URL` 是公开可访问的，任何人拿到该地址后理论上都可以向该链接推送消息，因此服务端需要解决两个问题：

1. 如何判断请求是否来自企业微信
2. 如何判断推送消息内容是否被篡改

---

## 2.3 数字签名校验

通过数字签名可以解决上述问题。

签名机制如下：

1. 开发者与企业微信约定 `Token` 作为密钥
2. `Token` 仅开发者和企业微信知道
3. `Token` 在传输过程中不可见
4. 企业微信推送消息时，会将消息内容与 `Token` 计算出签名
5. 开发者收到消息后，也按相同算法重新计算签名
6. 如果签名一致，则说明：

   * 来源可信
   * 内容完整

---

### 非企业微信来源的情况

如果请求不是来自企业微信：

* 攻击者没有正确的 `Token`
* 因此无法计算出正确签名

---

### 消息被篡改的情况

如果消息内容被篡改：

* 开发者会使用收到的消息内容和 `Token` 重新计算签名
* 重新计算出的签名会与参数中的签名不一致
* 因此应拒绝该请求

---

## 2.4 EncodingAESKey

`EncodingAESKey` 用于消息内容加密。

要求：

* 由英文或数字组成
* 长度为 43 位
* 由开发者自定义

由于消息在公开互联网中传输，内容可能被截获。

如果消息未加密：

* 截获者可以直接读取消息内容
* 若包含敏感信息，会存在较大风险

因此需要使用 `EncodingAESKey` 对消息内容进行加密，并按指定格式组装后发送。

---

# 3. 回调服务需要实现哪些功能

配置回调服务时，需要同时支持：

* `HTTP GET`
* `HTTP POST`

---

## 3.1 整体流程

企业微信会先判断 `URL` 服务是否具备解析企业微信推送消息的能力。

验证方式：

1. 企业微信向 `URL` 服务发送一条 `GET` 请求
2. 请求中携带签名与密文参数
3. `URL` 服务检查签名
4. 签名通过后，解密密文参数
5. 正确返回密文对应的明文字符串
6. 验证通过后，企业微信中的配置开始生效

后续业务请求，例如：

* 应用菜单点击事件
* 用户消息

也会以类似方式：

```text id="sim9co"
签名 + 密文
```

向服务 `URL` 推送消息。

`URL` 服务验证签名通过后，需要解密 `POST` 数据，得到业务消息明文。

---

# 3.1 支持 HTTP GET 请求验证 URL 有效性

假设企业接收消息的 URL 设置为：

```text id="94nvmp"
https://api.3dept.com
```

企业管理员保存回调配置时，企业微信会发送一条验证消息到该 URL。

---

## 请求方式

```text id="hr2665"
GET
```

---

## 请求地址

```text id="eg6m5d"
https://api.3dept.com/?msg_signature=ASDFQWEXZCVAQFASDFASDFSS&timestamp=13500001234&nonce=123412323&echostr=ENCRYPT_STR
```

---

## 参数说明

| 参数              | 类型      | 说明                                                                                  |
| --------------- | ------- | ----------------------------------------------------------------------------------- |
| `msg_signature` | String  | 企业微信加密签名，计算时结合了企业填写的 `token`、请求中的 `timestamp`、`nonce`、加密消息体                         |
| `timestamp`     | Integer | 时间戳，与 `nonce` 结合使用，用于防止请求重放攻击                                                       |
| `nonce`         | String  | 随机数，与 `timestamp` 结合使用，用于防止请求重放攻击                                                   |
| `echostr`       | String  | 加密字符串，需要解密得到消息内容明文；解密后包含 `random`、`msg_len`、`msg`、`receiveid` 四个字段，其中 `msg` 为消息内容明文 |

---

## URL 验证处理步骤

回调服务需要正确响应才能通过 URL 验证。

处理步骤如下：

1. 解析请求参数：

   * `msg_signature`
   * `timestamp`
   * `nonce`
   * `echostr`

   参数值需要做 `UrlDecode` 处理。

2. 根据已有的 `token`，结合以下参数重新计算签名：

   * `timestamp`
   * `nonce`
   * `echostr`

   然后与请求参数中的 `msg_signature` 比较，确认调用者合法性。

3. 解密 `echostr` 参数，得到消息内容，即 `msg` 字段。

4. 在 **1 秒内** 响应 `GET` 请求。

响应内容要求：

* 返回上一步得到的明文消息内容
* 不能加引号
* 不能带 BOM 头
* 不能带换行符

---

## 调试方式

步骤 2～3 可以直接使用验证 URL 函数完成。

也可以使用企业微信接口调试工具进行调试：

```text id="f1sqhs"
接口类型：建立连接
接口列表：测试回调模式
```

---

# 3.2 支持 HTTP POST 请求接收业务数据

假设企业接收消息的 URL 设置为：

```text id="j22jvd"
https://api.3dept.com
```

当用户触发回调行为时，企业微信会向该 URL 发送回调消息。

---

## 请求方式

```text id="yci9an"
POST
```

---

## 请求地址

```text id="oifp7b"
https://api.3dept.com/?msg_signature=ASDFQWEXZCVAQFASDFASDFSS&timestamp=13500001234&nonce=123412323
```

---

## 接收数据格式

```xml id="g6sgke"
<xml>
  <ToUserName><![CDATA[toUser]]></ToUserName>
  <AgentID><![CDATA[toAgentID]]></AgentID>
  <Encrypt><![CDATA[msg_encrypt]]></Encrypt>
</xml>
```

---

## 参数说明

| 参数              | 类型      | 说明                                                           |
| --------------- | ------- | ------------------------------------------------------------ |
| `msg_signature` | String  | 企业微信加密签名，结合了企业填写的 `token`、请求中的 `timestamp`、`nonce` 参数、加密的消息体 |
| `timestamp`     | Integer | 时间戳，与 `nonce` 结合使用，用于防止请求重放攻击                                |
| `nonce`         | String  | 随机数，与 `timestamp` 结合使用，用于防止请求重放攻击                            |
| `ToUserName`    | String  | 企业微信的 `CorpID`；当为第三方应用回调事件时，内容为 `suiteid`                    |
| `AgentID`       | String  | 接收的应用 ID，可在应用设置页面获取；仅应用相关回调会带该字段                             |
| `Encrypt`       | String  | 消息结构体加密后的字符串                                                 |

---

## 企业收到消息后的处理步骤

企业收到消息后，需要执行以下处理：

1. 校验 `msg_signature`
2. 解密 `Encrypt`，得到明文消息结构体
3. 如果需要被动回复消息，构造被动响应包
4. 正确响应本次请求

---

## 超时与重试机制

企业微信服务器在 **5 秒内** 收不到响应会断开连接，并重新发起请求。

重试规则：

```text id="r4mxyx"
总共重试 3 次
```

仅以下情况会触发重试：

* 网络连接失败
* 网络请求超时

建议：

* 接收回调后立即应答
* 业务逻辑异步处理

注意：

* 目前无法保证 100% 回调成功
* 如果开发者服务失败过多或超时过多，企业微信可能根据回调情况丢弃部分回调事件
* 不建议强依赖回调
* 建议额外建立机制对齐相关业务数据

---

## 被动回复说明

步骤 1～2 可以直接使用解密函数完成。

步骤 3 包含：

1. 加密被动回复消息
2. 生成新签名
3. 构造被动响应包

这些可以直接使用加密函数完成。

步骤 4 中，不同业务回调要求返回不同内容，例如：

* 空字符串
* 特定字符串，如 `success`
* 加密后的被动回复消息

具体要求需参考各业务回调文档。

---

## 被动响应包数据格式

```xml id="zw0zh4"
<xml>
  <Encrypt><![CDATA[msg_encrypt]]></Encrypt>
  <MsgSignature><![CDATA[msg_signature]]></MsgSignature>
  <TimeStamp>timestamp</TimeStamp>
  <Nonce><![CDATA[nonce]]></Nonce>
</xml>
```

---

## 被动响应包参数说明

| 参数             | 是否必须 | 说明          |
| -------------- | ---- | ----------- |
| `Encrypt`      | 是    | 经过加密的消息结构体  |
| `MsgSignature` | 是    | 消息签名        |
| `TimeStamp`    | 是    | 时间戳         |
| `Nonce`        | 是    | 随机数，由企业自行生成 |

---

# 3.3 获取企业微信服务器的 IP 段

企业微信在回调企业指定 URL 时，会通过特定 IP 发送请求。

如果企业需要配置防火墙，可以通过该接口获取所有相关 IP 段。

注意：

* IP 段可能发生变更
* 当 IP 段变更时，新旧 IP 段会同时保留一段时间
* 建议企业每天定时拉取 IP 段
* 并更新防火墙设置，避免因 IP 段变化导致网络不通

---

## 请求方式

```text id="tbx6kn"
GET（HTTPS）
```

---

## 请求地址

```text id="ih3dip"
https://qyapi.weixin.qq.com/cgi-bin/getcallbackip?access_token=ACCESS_TOKEN
```

---

## 请求参数说明

| 参数             | 必须 | 说明     |
| -------------- | -- | ------ |
| `access_token` | 是  | 调用接口凭证 |

---

## 权限说明

无限定。

---

## 返回结果

```json id="mi1332"
{
  "ip_list": [
    "1.2.3.4",
    "2.3.3.3"
  ],
  "errcode": 0,
  "errmsg": "ok"
}
```

---

## 返回参数说明

| 参数        | 类型          | 说明                        |
| --------- | ----------- | ------------------------- |
| `ip_list` | StringArray | 企业微信服务器 IP 段              |
| `errcode` | int         | 错误码，`0` 表示成功，非 `0` 表示调用失败 |
| `errmsg`  | string      | 错误信息，调用失败时返回相关错误信息        |

---

## access_token 过期返回示例

```json id="dxqg34"
{
  "ip_list": [],
  "errcode": 42001,
  "errmsg": "access_token expired, hint: [1576065934_28_e0fae07666aa64636023c1fa7e8f49a4], from ip: 9.30.0.138, more info at https://open.work.weixin.qq.com/devtool/query?e=42001"
}
```

判断方式：

```text id="q6wtls"
errcode != 0 表示调用失败
```
