# 获取日程详情

最后更新：2025/05/20

该接口用于获取指定的日程详情。

---

## 请求信息

* **请求方式**：`POST`（HTTPS）
* **请求地址**：

```text
https://qyapi.weixin.qq.com/cgi-bin/oa/schedule/get?access_token=ACCESS_TOKEN
```

---

## 请求包体

```json
{
  "schedule_id_list": [
    "17c7d2bd9f20d652840f72f59e796AAA"
  ]
}
```

---

## 请求参数说明

| 参数                 | 是否必须 | 类型       | 说明                     |
| ------------------ | ---- | -------- | ---------------------- |
| `schedule_id_list` | 是    | string[] | 日程 ID 列表，一次最多拉取 1000 条 |

---

## 返回参数说明

| 参数              | 类型     | 说明    |
| --------------- | ------ | ----- |
| `errcode`       | int32  | 返回码   |
| `errmsg`        | string | 错误码描述 |
| `schedule_list` | obj[]  | 日程列表  |

---

## schedule_list 字段说明

| 参数             | 类型       | 说明                  |
| -------------- | -------- | ------------------- |
| `schedule_id`  | string   | 日程 ID               |
| `admins`       | string[] | 管理员 userid 列表       |
| `attendees`    | obj[]    | 日程参与者列表             |
| `summary`      | string   | 日程标题                |
| `description`  | string   | 日程描述                |
| `reminders`    | obj      | 提醒相关信息              |
| `location`     | string   | 日程地址，不多于 128 个字符    |
| `cal_id`       | string   | 日程所属日历 ID，不多于 64 字节 |
| `start_time`   | uint32   | 日程开始时间，Unix 时间戳     |
| `end_time`     | uint32   | 日程结束时间，Unix 时间戳     |
| `status`       | uint32   | 日程状态                |
| `is_whole_day` | uint32   | 是否全天日程，`0`-否；`1`-是  |

---

## attendees 字段说明

| 参数                | 类型     | 说明         |
| ----------------- | ------ | ---------- |
| `userid`          | string | 日程参与者 ID   |
| `response_status` | uint32 | 日程参与者的接受状态 |

### response_status 取值

| 值   | 说明    |
| --- | ----- |
| `0` | 未处理   |
| `1` | 待定    |
| `2` | 全部接受  |
| `3` | 仅接受一次 |
| `4` | 拒绝    |

---

## reminders 字段说明

| 参数                         | 类型       | 说明                  |
| -------------------------- | -------- | ------------------- |
| `is_remind`                | int32    | 是否需要提醒。`0`-否；`1`-是  |
| `is_repeat`                | int32    | 是否重复日程。`0`-否；`1`-是  |
| `remind_before_event_secs` | uint32   | 日程开始前多少秒提醒          |
| `remind_time_diffs`        | int32[]  | 提醒时间与日程开始时间的差值      |
| `repeat_type`              | uint32   | 重复类型                |
| `repeat_until`             | uint32   | 重复结束时间，Unix 时间戳     |
| `is_custom_repeat`         | uint32   | 是否自定义重复。`0`-否；`1`-是 |
| `repeat_interval`          | uint32   | 重复间隔                |
| `repeat_day_of_week`       | uint32[] | 每周周几重复              |
| `repeat_day_of_month`      | uint32[] | 每月哪几天重复             |
| `timezone`                 | uint32   | 时区                  |
| `exclude_time_list`        | obj[]    | 重复日程不包含的日期列表        |

---

## remind_before_event_secs 说明

> 建议使用 `remind_time_diffs` 字段，`remind_before_event_secs` 后续将会废弃。

支持值：

| 值       | 说明          |
| ------- | ----------- |
| `0`     | 事件开始时       |
| `300`   | 事件开始前 5 分钟  |
| `900`   | 事件开始前 15 分钟 |
| `3600`  | 事件开始前 1 小时  |
| `86400` | 事件开始前 1 天   |

---

## remind_time_diffs 说明

`remind_time_diffs` 表示提醒时间与 `start_time` 的差值。

例如：

```text
-300
```

表示：

```text
日程开始前 5 分钟提醒
```

特殊情况：

企业微信终端设置的“全天”类型日程，由于 `start_time` 是 0 点时间戳，如果提醒设置为当天 9 点，则会出现正数：

```text
32400
```

取值范围：

```text
-604800 ~ 86399
```

---

## repeat_type 取值

| 值   | 说明  |
| --- | --- |
| `0` | 每日  |
| `1` | 每周  |
| `2` | 每月  |
| `5` | 每年  |
| `7` | 工作日 |

---

## 自定义重复字段说明

### repeat_interval

仅当自定义重复时有效。

该字段含义会随 `repeat_type` 不同而变化。

例如：

| 配置                         | 含义         |
| -------------------------- | ---------- |
| `repeat_interval=2` + 每周重复 | 每 2 周重复一次  |
| `repeat_interval=2` + 每月重复 | 每 2 个月重复一次 |

---

### repeat_day_of_week

仅当：

```text
is_custom_repeat = 1
```

且重复类型为每周时有效。

取值范围：

```text
1 ~ 7
```

分别表示：

```text
周一 ~ 周日
```

---

### repeat_day_of_month

仅当：

```text
is_custom_repeat = 1
```

且重复类型为每月时有效。

取值范围：

```text
1 ~ 31
```

分别表示每月 `1 ~ 31` 号。

---

## timezone 时区说明

使用 UTC 偏移量表示：

* 东区为正数
* 西区为负数

例如：

```text
+8
```

表示北京时间东八区。

取值范围：

```text
-12 ~ +12
```

默认值为北京时间东八区。

---

## exclude_time_list 字段说明

`exclude_time_list` 表示重复日程不包含的日期列表。

当对重复日程修改或删除特定一天或多天时，原日程会排除对应日期。

| 参数                             | 类型     | 说明        |
| ------------------------------ | ------ | --------- |
| `exclude_time_list.start_time` | uint32 | 不包含的日期时间戳 |

---

## status 日程状态

| 值   | 说明  |
| --- | --- |
| `0` | 正常  |
| `1` | 已取消 |

---

## 注意事项

被取消的日程也可以拉取详情，调用者需要检查：

```text
status
```

判断日程是否已取消。
