# 获取日历下的日程列表

最后更新：2022/12/06

该接口用于获取指定日历下的日程列表。

> 仅可获取应用自己创建的日历下的日程。

---

## 请求信息

* **请求方式**：`POST`（HTTPS）
* **请求地址**：

```text id="w4n7kp"
https://qyapi.weixin.qq.com/cgi-bin/oa/schedule/get_by_calendar?access_token=ACCESS_TOKEN
```

---

## 请求包体

```json id="x2m5ra"
{
  "cal_id": "wcjgewCwAAqeJcPI1d8Pwbjt7nttzAAA",
  "offset": 100,
  "limit": 1000
}
```

---

## 参数说明

| 参数       | 是否必须 | 类型     | 说明                          |
| -------- | ---- | ------ | --------------------------- |
| `cal_id` | 是    | string | 日历 ID                       |
| `offset` | 否    | uint32 | 分页偏移量，默认 `0`                |
| `limit`  | 否    | uint32 | 分页数量，默认 `500`，范围 `1 ~ 1000` |

---

# 分页说明

当日程较多时，需要使用：

* `offset`
* `limit`

进行分页拉取。

---

## offset 规则

`offset`：

```text id="t7m2ow"
从 0 开始
```

例如：

| offset | limit | 含义              |
| ------ | ----- | --------------- |
| `0`    | `100` | 获取第 1 ~ 100 条   |
| `100`  | `100` | 获取第 101 ~ 200 条 |
| `200`  | `100` | 获取第 201 ~ 300 条 |

---

## 分页终止条件

当返回：

```text id="f8y9ql"
schedule_list = []
```

时表示：

```text id="p1j4vh"
offset 已超过可获取范围
```

此时应停止继续拉取。

---

## 增量获取

如果后续有新增日程：

```text id="r6s2na"
可继续基于当前 offset 增量拉取
```

# 返回参数说明

| 参数              | 类型     | 说明    |
| --------------- | ------ | ----- |
| `errcode`       | int32  | 返回码   |
| `errmsg`        | string | 错误码描述 |
| `schedule_list` | obj[]  | 日程列表  |

---

# schedule_list 字段说明

| 参数            | 类型       | 说明                 |
| ------------- | -------- | ------------------ |
| `schedule_id` | string   | 日程 ID              |
| `sequence`    | uint64   | 日程编号，自增数字          |
| `admins`      | string[] | 管理员 userid 列表      |
| `attendees`   | obj[]    | 日程参与者列表，最多支持 300 人 |
| `summary`     | string   | 日程标题               |
| `description` | string   | 日程描述               |
| `reminders`   | obj      | 提醒相关信息             |
| `location`    | string   | 日程地址               |
| `start_time`  | uint32   | 开始时间（Unix 时间戳）     |
| `end_time`    | uint32   | 结束时间（Unix 时间戳）     |
| `status`      | uint32   | 日程状态               |
| `cal_id`      | string   | 所属日历 ID            |

---

# attendees 字段说明

| 参数                | 类型     | 说明     |
| ----------------- | ------ | ------ |
| `userid`          | string | 参与者 ID |
| `response_status` | uint32 | 参与状态   |

---

## response_status 取值

| 值   | 说明    |
| --- | ----- |
| `0` | 未处理   |
| `1` | 待定    |
| `2` | 全部接受  |
| `3` | 仅接受一次 |
| `4` | 拒绝    |

---

# reminders 字段说明

| 参数                         | 类型       | 说明      |
| -------------------------- | -------- | ------- |
| `is_remind`                | int32    | 是否提醒    |
| `is_repeat`                | int32    | 是否重复日程  |
| `remind_before_event_secs` | uint32   | 提前提醒秒数  |
| `repeat_type`              | uint32   | 重复类型    |
| `repeat_until`             | uint32   | 重复结束时间  |
| `is_custom_repeat`         | uint32   | 是否自定义重复 |
| `repeat_interval`          | uint32   | 重复间隔    |
| `repeat_day_of_week`       | uint32[] | 每周重复日期  |
| `repeat_day_of_month`      | uint32[] | 每月重复日期  |
| `timezone`                 | uint32   | 时区      |

---

# repeat_type 取值

| 值   | 说明  |
| --- | --- |
| `0` | 每日  |
| `1` | 每周  |
| `2` | 每月  |
| `5` | 每年  |
| `7` | 工作日 |

---

# 时区 timezone

使用 UTC 偏移量表示：

| 示例   | 含义          |
| ---- | ----------- |
| `+8` | 北京时间（UTC+8） |

取值范围：

```text id="q9m3za"
-12 ~ +12
```

默认值：

```text id="y4t8ov"
UTC+8
```

---

# status 日程状态

| 值   | 说明  |
| --- | --- |
| `0` | 正常  |
| `1` | 已取消 |

---

# 注意事项

即使：

```text id="e8u1mx"
status = 1
```

即日程已取消：

```text id="j2s5pa"
仍然可以获取该日程详情
```

因此调用方需要主动检查：

```text id="d5v0ln"
status
```

字段判断日程状态。
